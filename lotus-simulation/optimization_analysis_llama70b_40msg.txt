Method
- Read each JSON's config.resolved_config, config.optimizations, operations[], and summary.
- Counted cascade activation per operator (cascade_enabled true/false), cascade_stats presence, fallback errors, and cache counters.
- Compared wall-clock runtime from start_time/end_time and cost fields from summary.

Important Accounting Caveat
- The script summaries read usage from lm.stats (main LM only), not helper_lm stats.
- Therefore, physical/virtual cost in these logs does NOT represent full helper-model spend.
- If helper_lm is used (e.g., vLLM), true total cost/throughput must also be read from helper-side telemetry (vLLM logs/metrics).

============================================================
C) Why zep still appears to "not use vLLM"
============================================================
1) Main model remained deepseek in the resolved config.
2) zep workload is dominated by sem_join, not sem_filter.
3) LOTUS join optimizer path is primarily similarity-proxy based and does not currently execute helper-lm as the join proxy path.
4) Therefore, even with proxy_model=helper_lm in config, zep join behavior is constrained by LOTUS sem_join implementation, not only YAML settings.

- Operator cache: used in mem0 and zep; absent in evermemos latest.

============================================================
E) Root Cause Attribution
============================================================
Workflow-driven
- mem0 and evermemos each have a sem_filter stage where helper cascade can be directly applied.
- zep insertion relies heavily on sem_join stages; behavior depends on LOTUS sem_join optimizer internals.

LOTUS-implementation-driven
- sem_join helper-lm proxy path is not the active default join helper mechanism; similarity proxy dominates.
- This limits direct vLLM-helper impact in zep join-heavy flows.

Script/data-driven
- zep Q5 shows recurring index-out-of-bounds fallback errors, reducing effective optimization and adding overhead.
- evermemos segmentation increased to 13 segments in the latest run, increasing downstream work.
